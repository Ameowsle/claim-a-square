# CLAUDE.md

This is a practice repo for our GitHub workshop. Rules for working here:

## Workshop pacing (important)

- When the user asks for a single step (create a branch, commit, push, open
  a pull request), do exactly that one step and **stop**. Do not chain ahead
  to the next steps — the workshop goes one step at a time, together.
- After each step, briefly say what just happened and where the user can see
  it (VS Code status bar, GitHub branch list, the PR page).

## Git rules

- Always work on a branch named after the user. Never commit or push to
  `main` — only the facilitator (Amelia) may do that.
- Never force-push. Never delete branches after merging (the network graph
  should keep them).
- Commit messages: short and imperative, e.g. "Add Henk's card".

## Editing rules

- Participants only edit what the facilitator assigned: their slot in
  `team.html`, or the specific line/file named in their task.
- Never edit other people's cards or slots.
